## 小结

以上这么多内容仅仅只涉及了前端开发的技术储备部分。

对于实际的业务和代码产出，还有无数细节和额外的套路，  
例如：兼容性、骨架屏、小程序、动画、表单、i18n…

不过当你的技能体系能覆盖以上至少一半内容时，  
那么你已经是一个不错的工程师了 :)
